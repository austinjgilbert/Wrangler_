# Repository Guardrails

This repository is not a generic AI assistant and not a generic research bot.

It is an action-first SDR execution engine.

## Primary Rule
Optimize for high-quality, evidence-backed, timely SDR actions.

Do not optimize for generic summaries, research completeness alone, or maximum automation for its own sake.

## Non-Negotiable Principles
- Evidence before inference
- Action prioritization over data accumulation
- Signal fusion over single-signal excitement
- Explainability over opaque scoring
- Learning with guardrails over uncontrolled adaptation
- Freshness and decay over stale certainty
- Strategy versioning over silent mutation
- Outcome linkage over vanity scoring
- Regression safety over prompt drift
- Superuser control over black-box behavior

## Evaluation Questions
Any architecture or code change should be evaluated against these questions:

1. Does this improve action quality?
2. Does this reduce operator thinking time?
3. Does this improve confidence calibration?
4. Does this preserve explainability?
5. Does this prevent stale or low-value accumulation?
6. Does this create a measurable learning signal?
7. Does this reduce drift risk?

## Required Mindset
The system should not just ask:
- What can I infer?

It should always ask:
- What can I justify?
- What action does this support?
- How fresh is this?
- What would prove me wrong?
- Am I getting better over time or just becoming more confident?
